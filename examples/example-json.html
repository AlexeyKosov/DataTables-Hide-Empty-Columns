The table below is a basic DataTables instance with hideEmptyCols configured to process two columns, which are targeted using the column name, as opposed to the column index. This is definitely a better method, especially if you generate the table columns dynamically. You may not want to hard code the column indexes, that would require you to change it every time the columns configuration gets changed. The hideEmptyCols configuration value can take an array of both column indexes, as well as the column names.
