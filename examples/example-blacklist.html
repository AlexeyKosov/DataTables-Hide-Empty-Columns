The table below is a basic DataTables instance with a slightly more advanced hideEmptyCols configuration. By default, hideEmptyCols will look at all columns, but you can override this by using the hideEmptyCols.columns option. This option can take the column indexes (integer), as well as the column names (string), as long as they are provided inside an array. By default, the hideEmptyCols.columns option will signify what columns will be processed, making it a white-list, but if you wish to use it to specify what columns to ignore, then you can set the hideEmptyCols.columns option to false, which set the plugin to process all other columns. The below example was configured to ignore the column with the index 3, which is the Office column, so notice that even though both the Office and Age columns have no values, only the Office column is hidden
